# Requirements Document: BharatChef AI Coach

## Introduction

BharatChef AI is an AI-powered cooking skill evaluation and coaching platform that objectively assesses culinary skills using video analysis. The system compares trainee cooking videos with expert benchmark videos and provides structured feedback, performance scores, and personalized coaching to help culinary students, trainee chefs, and aspiring cooks improve their skills systematically.

The MVP focuses on core video capture, action recognition, benchmark comparison, scoring, and AI-generated feedback for 3-5 initial dishes, deployable on low-cost Android smartphones with cloud processing.

## Glossary

- **System**: The BharatChef AI Coach platform
- **Trainee**: A user whose cooking skills are being evaluated (culinary student, junior cook, home entrepreneur)
- **Expert_Video**: A benchmark cooking video demonstrating the correct technique for a dish
- **Trainee_Video**: A video recorded by the trainee performing the cooking task
- **Dish_Metadata**: Information about a dish including name, duration, ingredients, and expected cooking steps
- **Performance_Score**: A numerical score (0-100) representing overall cooking performance
- **Category_Score**: Individual scores for specific aspects (step sequence, timing, technique, heat control, visual quality, plating)
- **AI_Coach**: The intelligent agent that interprets CV outputs and generates personalized coaching feedback
- **Action_Recognition**: Computer vision process that identifies cooking actions in video frames
- **DTW**: Dynamic Time Warping algorithm used for temporal alignment of cooking sequences
- **Benchmark_Comparison**: The process of comparing trainee performance against expert video
- **Coaching_Feedback**: Personalized, human-like suggestions for improvement generated by the AI Coach
- **Skill_Progression**: Tracking of improvement across multiple attempts at the same dish
- **CV_Model**: Computer Vision model (SlowFast, I3D, YOLOv8, ResNet) used for video analysis
- **Flame_Detection**: Computer vision process that identifies flame presence and intensity using brightness spectrum analysis
- **Heat_Intensity**: Estimated temperature level based on visual cues (steam, bubbling, color changes)

## Requirements

### Requirement 1: Video Capture and Upload

**User Story:** As a trainee, I want to record my cooking process using my smartphone, so that the system can evaluate my skills.

#### Acceptance Criteria

1. WHEN a trainee initiates video recording, THE System SHALL capture video from the smartphone's overhead camera
2. WHEN recording is in progress, THE System SHALL display real-time recording indicators (duration, storage space)
3. WHEN a trainee stops recording, THE System SHALL save the video locally on the device
4. WHEN a trainee uploads a video, THE System SHALL transfer the Trainee_Video to cloud storage
5. WHEN upload is in progress, THE System SHALL display upload progress percentage
6. WHEN upload completes, THE System SHALL confirm successful upload and provide a unique video identifier
7. IF network connectivity is lost during upload, THEN THE System SHALL pause upload and resume when connectivity is restored
8. THE System SHALL accept video files in MP4, MOV, or AVI formats
9. THE System SHALL validate that uploaded videos are between 30 seconds and 30 minutes in duration
10. WHERE a side camera is available, THE System SHALL support optional side camera video upload for heat and flame analysis

### Requirement 2: Dish Metadata Management

**User Story:** As a system administrator, I want to define dish metadata and associate expert videos, so that trainees can be evaluated against proper benchmarks.

#### Acceptance Criteria

1. WHEN an administrator creates a new dish entry, THE System SHALL store Dish_Metadata including name, expected duration, ingredients list, and expected cooking steps
2. WHEN an administrator uploads an Expert_Video, THE System SHALL associate it with the corresponding Dish_Metadata
3. THE System SHALL validate that Expert_Video contains all expected cooking steps defined in Dish_Metadata
4. WHEN a trainee selects a dish to practice, THE System SHALL retrieve the associated Dish_Metadata and Expert_Video
5. THE System SHALL maintain a catalog of at least 3-5 dishes for MVP scope

### Requirement 3: Action Recognition

**User Story:** As the system, I want to identify cooking actions in trainee videos, so that I can compare them with expert techniques.

#### Acceptance Criteria

1. WHEN a Trainee_Video is processed, THE System SHALL apply Action_Recognition using SlowFast or I3D models to detect cooking actions
2. WHEN processing video frames, THE System SHALL identify actions including chopping, stirring, flipping, pouring, and plating
3. WHEN an action is detected, THE System SHALL record the action type, start timestamp, end timestamp, and confidence score
4. THE System SHALL achieve at least 85% accuracy in Action_Recognition against ground truth labels
5. WHEN processing completes, THE System SHALL generate a structured action sequence list with timestamps

### Requirement 4: Ingredient and Object Detection

**User Story:** As the system, I want to detect ingredients and utensils in cooking videos, so that I can verify correct ingredient usage and technique.

#### Acceptance Criteria

1. WHEN a Trainee_Video is processed, THE System SHALL apply YOLOv8 object detection to identify ingredients in video frames
2. WHEN detecting ingredients, THE System SHALL identify common cooking ingredients including vegetables, proteins, spices, and liquids
3. THE System SHALL achieve at least 80% accuracy in ingredient detection against ground truth labels
4. WHEN detecting objects, THE System SHALL identify cooking utensils including knives, pans, spatulas, bowls, and plates
5. WHEN an ingredient or object is detected, THE System SHALL record the object type, bounding box coordinates, timestamp, and confidence score
6. WHEN detection completes, THE System SHALL generate an ingredient usage report comparing detected ingredients with expected ingredients from Dish_Metadata
7. THE System SHALL flag missing ingredients or incorrect ingredient substitutions

### Requirement 5: Heat and Flame Analysis

**User Story:** As the system, I want to analyze heat and flame control in cooking videos, so that I can evaluate temperature management skills.

#### Acceptance Criteria

1. WHERE a side camera video is provided, THE System SHALL analyze flame and heat patterns throughout the cooking process
2. WHEN analyzing flames, THE System SHALL apply brightness spectrum analysis to detect flame presence and intensity
3. WHEN flame is detected, THE System SHALL classify flame level as low, medium, or high based on brightness and color spectrum
4. WHEN estimating heat intensity, THE System SHALL analyze visual cues including steam, bubbling, and color changes in food
5. WHEN comparing with Expert_Video, THE System SHALL identify heat control deviations including overheating, underheating, or incorrect flame timing
6. THE System SHALL generate a heat control score based on alignment with expert heat patterns
7. IF side camera video is not provided, THEN THE System SHALL estimate heat control from overhead camera visual cues only

### Requirement 6: Temporal Sequence Alignment

**User Story:** As the system, I want to align trainee cooking sequences with expert sequences, so that I can identify step deviations and timing differences.

#### Acceptance Criteria

1. WHEN comparing Trainee_Video with Expert_Video, THE System SHALL apply DTW algorithm to align the two action sequences
2. WHEN alignment is complete, THE System SHALL identify missing steps, extra steps, and out-of-order steps
3. WHEN alignment is complete, THE System SHALL calculate timing differences for each corresponding step
4. THE System SHALL generate a step deviation report listing all sequence discrepancies
5. THE System SHALL calculate step sequence accuracy as a percentage of correctly performed steps in correct order

### Requirement 7: Visual Quality Analysis

**User Story:** As the system, I want to compare visual appearance of trainee dishes with expert dishes, so that I can evaluate presentation quality.

#### Acceptance Criteria

1. WHEN analyzing final dish appearance, THE System SHALL extract visual embeddings using ResNet from both Trainee_Video and Expert_Video final frames
2. WHEN embeddings are extracted, THE System SHALL calculate cosine similarity between trainee and expert dish embeddings
3. THE System SHALL analyze color distribution, texture patterns, and plating arrangement
4. WHEN visual analysis completes, THE System SHALL generate a visual similarity score (0-100)
5. THE System SHALL identify specific visual differences (color variations, texture issues, plating gaps)

### Requirement 8: Performance Scoring

**User Story:** As a trainee, I want to receive an objective performance score, so that I can understand my current skill level.

#### Acceptance Criteria

1. WHEN all analysis components complete, THE System SHALL calculate an overall Performance_Score (0-100)
2. THE System SHALL calculate Category_Score for each category: step sequence accuracy, timing accuracy, technique quality, heat control, visual quality, and plating quality
3. WHEN calculating scores, THE System SHALL apply weighted aggregation with explainable logic
4. THE System SHALL assign weights: step sequence (25%), timing (20%), technique (20%), visual quality (20%), heat control (10%), plating (5%)
5. WHEN scores are calculated, THE System SHALL store them with timestamp and video identifier for tracking
6. THE System SHALL determine skill certification level (Beginner: 0-50, Intermediate: 51-75, Advanced: 76-100)

### Requirement 9: AI Coach Feedback Generation

**User Story:** As a trainee, I want to receive personalized coaching feedback, so that I know exactly how to improve my cooking skills.

#### Acceptance Criteria

1. WHEN CV_Model outputs are available, THE AI_Coach SHALL interpret action recognition results, timing data, visual analysis, ingredient detection, object detection, and heat control data
2. WHEN interpreting CV outputs, THE AI_Coach SHALL use multimodal LLM to analyze video frames, detected objects, action sequences, and scoring data
3. WHEN generating feedback, THE AI_Coach SHALL identify the top 3-5 most critical mistakes or improvement areas
4. WHEN mistakes are identified, THE AI_Coach SHALL generate human-like Coaching_Feedback explaining what went wrong and why
5. WHEN generating suggestions, THE AI_Coach SHALL provide specific corrective actions for each identified issue
6. THE AI_Coach SHALL prioritize feedback based on impact on overall performance
7. THE AI_Coach SHALL generate contextually relevant and encouraging feedback that maintains trainee motivation
8. WHEN feedback is generated, THE System SHALL present it in structured format with mistake description, impact, and suggested correction

### Requirement 10: Results Dashboard Display

**User Story:** As a trainee, I want to view my evaluation results on a dashboard, so that I can review my performance and track progress.

#### Acceptance Criteria

1. WHEN evaluation completes, THE System SHALL display overall Performance_Score prominently on the dashboard
2. WHEN displaying results, THE System SHALL show Category_Score breakdown with visual indicators (charts or progress bars)
3. WHEN displaying feedback, THE System SHALL present AI_Coach Coaching_Feedback in readable, organized format
4. WHEN displaying analysis, THE System SHALL show step deviation report with timestamp-based feedback
5. THE System SHALL display skill certification level with visual badge or indicator
6. WHEN a trainee has multiple attempts, THE System SHALL display Skill_Progression chart showing score trends over time
7. THE System SHALL allow trainees to view side-by-side comparison of their video with Expert_Video

### Requirement 11: Skill Progression Tracking

**User Story:** As a trainee, I want to track my improvement over multiple attempts, so that I can see my skill development over time.

#### Acceptance Criteria

1. WHEN a trainee completes multiple attempts for the same dish, THE System SHALL store all Performance_Score values with timestamps
2. WHEN displaying progression, THE System SHALL calculate improvement percentage between attempts
3. THE System SHALL identify repeated mistakes across attempts
4. WHEN repeated mistakes are detected, THE AI_Coach SHALL emphasize these issues in feedback
5. THE System SHALL track whether specific mistakes decrease by at least 30% after 3 attempts
6. THE System SHALL calculate overall skill improvement and display it as percentage change

### Requirement 12: System Performance and Scalability

**User Story:** As a system operator, I want the platform to process evaluations efficiently and cost-effectively, so that it can scale to many users.

#### Acceptance Criteria

1. WHEN processing a Trainee_Video, THE System SHALL complete full evaluation within 5 minutes for a 10-minute video
2. THE System SHALL process videos using cloud-based compute resources
3. THE System SHALL target cost per evaluation of less than â‚¹2
4. WHEN multiple users submit videos simultaneously, THE System SHALL queue and process them without blocking
5. THE System SHALL support deployment on low-cost Android smartphones (no specialized hardware required)
6. THE System SHALL optimize CV_Model inference using batch processing or model quantization where applicable

### Requirement 13: Error Handling and Validation

**User Story:** As a trainee, I want the system to handle errors gracefully, so that I receive clear guidance when something goes wrong.

#### Acceptance Criteria

1. IF a Trainee_Video upload fails, THEN THE System SHALL display a clear error message and allow retry
2. IF Action_Recognition confidence is below 60% for critical steps, THEN THE System SHALL flag low-confidence detections in the report
3. IF a Trainee_Video does not match the selected dish, THEN THE System SHALL notify the trainee and request correct video
4. IF Expert_Video is missing for a selected dish, THEN THE System SHALL prevent evaluation and notify administrator
5. IF cloud processing fails, THEN THE System SHALL log the error, notify the trainee, and provide option to retry
6. THE System SHALL validate video format, duration, and file size before processing
7. IF validation fails, THEN THE System SHALL provide specific error message indicating the issue

### Requirement 14: Data Privacy and Security

**User Story:** As a trainee, I want my cooking videos and performance data to be secure, so that my privacy is protected.

#### Acceptance Criteria

1. WHEN a trainee uploads a video, THE System SHALL encrypt the video during transmission using TLS
2. WHEN storing videos, THE System SHALL store them in secure cloud storage with access controls
3. THE System SHALL ensure that trainees can only access their own videos and performance data
4. WHEN a trainee requests data deletion, THE System SHALL remove all associated videos and performance records within 30 days
5. THE System SHALL not share trainee data with third parties without explicit consent
6. THE System SHALL comply with applicable data protection regulations
